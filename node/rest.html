Currently implemented REST API

/networks - all networks
/networks/turneronly - only Turner networks
/networks/notturner - all networks but Turner networks
/networks/:id - network by id
/schedules - all schedules
/schedules/:id - schedule by id
/schedules/platform/:platform - schedules by platform
/schedules/source/:source - schedules by source
/schedules/network/:network - schedules by network
/schedules/network/:network/today - schedule by network for today
/schedules/network/:network/startdate/:startdate - schedule by network for startdate
/schedules/startdate/:startdate/enddate/:enddate - schedule for range between start and end date
/schedules/network/turneronly/today - schedules for turner only networks for today
